CRACTERÍSTICAS DE UM BANCO DE DADOS:

1-NATUREZA DE AUTODESCRIÇÃO DE UM SISTEMA DE BANCO DE DADOS:

O SGBD CONTEM O BANCO DE DADOS E TAMBÉM UMA DEFINIÇÃO OU DESCRIÇÃO COMPLETA DA SUA ESTRUTURA E RESTRIÇÕES

A DEFINIÇÃO OU DESCRIÇÃO DE UM BANCO DE DADOS É ARMAZENADO NO CATÁLOGO DO SGBD

E ESSA MESMA DEFINIÇÃO DE DADOS ARMAZENADA NO CATÁLOGO É CHAMADA DE METADADOS

O METADADOS DEFINE A ESTRUTURA DE CADA  ARQUIVO OU DADOS
O TIPO E FORMATO DE ARMAZENAMENTO DOS ARQUIVOS OU DADOS
E A RESTRIÇÃO SOBRE OS DADOS


O CATÁLOGO ARMAZENA OS METADADOS E OS METADADOS DEFINEM OU DESCREVE ESSES DADOS ARMAZENADOS DA SEGUINTE MANEIRA: 

A ESTRUTURA DE CADA ARQUIVO
O TIPO E O FORMATO DE ARMAZENAMENTO DE CADA ARQUIVO
A RESTRIÇÃO SOBRE OS DADOS

DADOS E A DESCRIÇÃO DAS ESTRUTURAS DE ARMAZENAMENTO DOS DADOS

2-ISOLAMENTO ENTRE PROGRAMAS E DADOS E ABSTRAÇÃO DE DADOS


COM O SGBD AS APLICAÇÕES QUE ACESSAM OS DADOS ELAS NÃO FAZEM ESSE ACESSO DE UMA FORMA DIREITA ELAS PRECISAM PRIMEIRAMENTE ACESSAR O SGBD QUE É A PONTE ENTRE A APLICAÇÃO E OS DADOS

OS DADOS ESTÃO ARMAZENADOS NO CATÁLOGO DO SGBD E NÃO NAS APLICAÇÕES QUE ACESSSAM ESSES DADOS

OU SEJA O SGBD TORNA OS DADOS INDEPENDENTES 

ABSTRAÇÃO DE DADOS:

O USUÁRIO NÃO PRECISA ENTENDERR COMO OS DADOS SÃO ARMAZENADOS QUAIS SÃO OS DETALHES DE ARMAZENAMENTO COMO SÃO IMPLEMENTADAS AS OPERAÇÕES SOBRE OS DADOS
